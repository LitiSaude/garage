Chapter 04 · API reference

API reference

Seven endpoints across BFF · orders · webhooks. All /commerce/v1/*, fronted by the AWS API Gateway at api.litisaude.com.br. The BFF lives in pygeia main; orders + webhooks live in liti-commerce — clients don't see the split. Customer auth via Clerk except for the two webhook routes (provider-specific signature verification).

BFF · prescribed-meds quick-buy

One endpoint, one screen.

Powers the Comprar card on Home and the prescription-detail screen. Lives in pygeia main (not liti-commerce) — composes prescription + buyable SKU + last-known price + default delivery address + default payment profile into one screen-shaped response. The missing[] contract drives mobile's CTA: [] → Comprar; ["payment_profile"] → "Adicionar cartão"; ["prescription_expired"] → "Renovar prescrição".

Webhooks

Provider-agnostic sinks.

New providers add only an adapter — router unchanged. Adding Drogasil, Pague Menos, etc. needs no code in this file.
