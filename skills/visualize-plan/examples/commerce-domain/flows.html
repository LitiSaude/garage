Chapter 02 · Flows

Flows

Charge first. Place second. No automated compensation. Two Temporal workflows on task queue commerce-default; five activities; two webhook resume points.

Forward-only saga

Charge first. Place second.

If place_order permanently fails after PAID, the order sits in PAID + FAILED, the stuck-orders cron emits a Datadog signal, and Ops handles refund manually via the Vindi dashboard.

Two workflows · five activities · two resume points · one manual-recovery state
Idempotency · 3 layers

Three layers, all mandatory. Temporal alone isn't enough.

Each layer guards a different failure mode. The DB guard is the primary defence — it covers the worst case (external system accepted, our DB write crashed before recording the id).

Three layers nested · Layer 1 (DB guard) is the primary defence

Layer 1 · DB guard

Always on. Activities re-read external_order_id / external_payment_id before calling external systems; non-null short-circuits.

Guards: the worst race — provider accepted, our DB write crashed before recording the id.

Layer 2 · Workflow uniqueness

WorkflowIdReusePolicy=REJECT_DUPLICATE with execution_id=str(order.id). Concurrent attempts get WorkflowAlreadyStartedError, caught + treated as success.

Guards: charge activity + Vindi webhook both wanting to start fulfill.

Layer 3 · Provider keys

Xis seller_order_id = str(order.id). Vindi bill.code = f"order-{order.id}". Used when honored.

Never relied on alone: sandbox vs prod, version drift make provider contracts loose.

Webhook lifecycle

Audit row first. State transition second. Always 204 (unless auth).

A Xis Type-1 webhook lifecycle, lane by lane. Audit-row writes happen in a separate transaction from state mutations — so a state-rollback never erases the audit.

Type 1 fulfillment webhook · 9-step lifecycle across 5 lanes

If audit-row write fails

Raise 500. Xis retries.

If state Tx fails after audit committed

Audit row already records what arrived + a processing_error. notice_error'd, return 204. Don't ask Xis to retry — operator replays manually.
