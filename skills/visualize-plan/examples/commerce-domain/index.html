01 · TL;DR

Tap Comprar on the GLP-1 card. Tap Confirmar. Done.

Today: GLP-1 prescription quick-buy. Tomorrow: lab tests, devices, supplements, generic catalog browse. The data model and internal seams are channel-agnostic from day one so future surface is additive — one row class in products, one seed script, one entry in the channel dispatch table. The v1 customer-facing surface is intentionally narrow: BFF + preview + create + list + detail + 2 webhooks.

Opaque routing principle
pharmacy_provider never appears in any client-visible response from BFF, preview, or order-create. It's a backend-internal decision. Order detail MAY surface a customer-friendly pharmacy_name display string post-placement — never the enum. JSON-grep regression tests assert no leaks.
Gateway-fronted
liti-commerce is internal-only on Fly's network. Mobile reaches it via api.litisaude.com.br/commerce/{proxy+} through the existing AWS API Gateway with X-Origin-Token enforcement. The BFF for the quick-buy card lives on pygeia main; webhook providers point at the gateway, not at Fly.
