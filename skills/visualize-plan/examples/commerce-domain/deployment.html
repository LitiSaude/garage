Chapter 05 · Deployment

Deployment topology

One monorepo. Two Fly.io apps behind one AWS API Gateway. Three connection strings. "Deployment-time decomposition, not service-level." Source stays unified; the process is separate; the public surface is shared. liti-commerce is internal-only on Fly's network — mobile reaches it via api.litisaude.com.br/commerce/{proxy+} with X-Origin-Token enforcement.

Topology

One gateway up front. Same monorepo at the top. Two apps in between. Shared infra at the bottom.

AWS API Gateway fronts both Fly apps · liti-commerce internal-only · BFF lives in pygeia main · shared monorepo + infrastructure
Cross-domain reads

Cross-domain reads

Standalone process holds 3 connection strings: COMMERCE_DATABASE_URL R/W + CONTINUOUS_CARE_DATABASE_URL RO + CUSTOMER_LIFECYCLE_DATABASE_URL RO.

Trade-off: tight runtime coupling — schema migrations in cont_care must stay backward-compatible until commerce redeploys. Same as today's monolith; only the process boundary is new.

Inter-process communication

Pub/Sub: commerce publishes pharmacy_order_* events; main pygeia subscribers consume.

Temporal: shared cluster, separate task queue (commerce-default).

HTTP commerce → main: none v1.

Phase-2 path

Swap direct DB access for HTTP RPC via a thin lib/internal_api_client/ behind an env flag. Decouples deploy cadence at the cost of latency.

Out of scope v1. Don't build now.

Migrations · CI/CD

Release-command runs Alembic before image goes live.

Migrations

Fly.io release_command runs alembic upgrade head against COMMERCE_DATABASE_URL before the new image goes live. Failure aborts the deploy.

Main pygeia does NOT run commerce migrations — they're owned by this app.

CI/CD

.github/workflows/deploy-commerce.yml triggers on push to main when commerce-relevant paths change. Steps: tests → lint → cross-domain-import check → build image → flyctl deploy.

Concurrency lock so two PRs landing fast don't race deploys.

Operational posture

Same on-call rotation as main pygeia, with separate Datadog monitor configured per liti-commerce service.

Logs: Fly.io → Datadog log shipping. Same redaction/PII rules.

Cost: ~$15-30/mo Fly.io for the two process types at min=1 each; right-sized after one week of staging traffic.
